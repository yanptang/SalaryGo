# M3：市场数据刷新

M3 定义统一请求和报价结构，支持 `stock`、`fund`、`index`、`nav` 和 `fx` 五类数据。每条成功数据必须包含标识、数值、币种、数据时点、抓取时点和来源。

刷新按单条请求隔离失败：某个标的失败时保留它之前的成功缓存，其他标的继续刷新，持仓价格不会被清空。缓存快照按数据类型的最大时效计算 `fresh` 或 `stale`，过期项同时生成显式警告。

当前实现包含可复现的离线 `StaticMarketDataProvider`，用于接口和失败场景验收。正式生产数据源尚未选择，不能把 fixture 数值当成真实行情。后续接入真实提供方时只需实现相同 provider 接口，并补授权、频率限制和来源条款。
