# M1：本地数据与用户档案

本地页面在档案或账本缺失时显示首次初始化向导。向导把用户填写的百分数转换为内部小数格式，并使用保守的默认集中度边界；高级用户仍可在“设置”中查看和修改结构化数据。

## 数据边界

- 示例档案位于 `examples/`，可以进入 Git，但不得包含真实个人信息。
- 真实档案默认位于 `data/private/profile.json`。
- 备份默认位于 `data/backups/`。
- 真实档案和备份目录均由 `.gitignore` 排除。
- `SALARYGO_DATA_DIR` 或 CLI 的 `--data-dir` 可指定隔离目录，自动化测试始终使用临时目录。

## 档案版本

- `schema_version`：数据结构版本。M1 仅接受版本 `1`，未知版本明确拒绝，避免误读。
- `revision`：用户档案修订号。首次保存为 `1`，每次保存或覆盖恢复加 `1`。
- `expected_revision`：可选的乐观锁。页面或 Agent 使用旧版本写入时会收到冲突错误。
- `created_at` 与 `updated_at`：由存储层统一生成带时区的 UTC 时间，调用方不能决定。

## 初始化字段

| 分组 | 字段 | 说明 |
| --- | --- | --- |
| 用户 | `display_name`, `base_currency` | 本地显示名和组合基准币种 |
| 投资计划 | `horizon_years`, `target_annual_return`, `max_drawdown` | 期限、目标和最大可接受回撤；百分比用小数表示 |
| 投资计划 | `monthly_investment`, `crash_behavior`, `allow_cash` | 常规月投入、下跌行为和是否允许保留现金 |
| 资金安全 | `emergency_fund`, `planned_uses` | 应急金状态及未来有明确用途的资金 |
| 渠道 | `accounts`, `funding_currencies` | 可用账户、市场、账户币种和投入币种 |
| 风险限制 | `concentration_limits` | 单只个股、个股合计、市场、行业和币种上限 |
| 排除项 | `exclusions` | 不投资的资产、行业、公司和具体产品 |

## 冲突规则

M1 明确阻断以下冲突，Agent 应据此继续追问，而不是自行猜测：

- 目标年化不低于 10%，但最大回撤低于 15%；
- 目标年化不低于 15%，但最大回撤低于 25%；
- 没有应急资金，同时禁止暂留现金；
- 单只个股上限高于个股合计上限；
- 常规投入币种不在投入币种列表中；
- 投入币种没有任何账户支持；
- 市场或币种上限与可用集合组合后数学上无法达到 100%。

这些是产品输入一致性规则，不代表对未来收益或回撤的预测。

## 写入安全

- 保存前先完成全量校验；失败时不改变现有档案。
- 使用同目录临时文件、`fsync` 和原子替换，减少中途退出导致的半写入。
- 档案和备份权限设为仅当前用户可读写（`0600`）。
- 恢复默认拒绝覆盖；`--replace` 也只允许覆盖相同 `profile_id` 的档案。
- 每份备份以 SHA-256 摘要命名，并生成包含完整摘要的 manifest。
